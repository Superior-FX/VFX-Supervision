## VFX Turnover Package

A **VFX turnover** is the formal handoff of VFX shots from editorial to VFX post.
It gives vendors everything needed to begin work.

### When Turnovers Happen
- **First turnover**: As soon as a substantial cut is available — don't wait for picture lock
- **Rolling turnovers**: As new shots are cut in or sequences are refined
- **Final turnover**: At or near picture lock — any shots added after this are change orders

### Turnover Package Contents
| Item | Format | Purpose |
|------|--------|---------|
| VFX Pull List | Spreadsheet (CSV or Excel) | Master list of all shots |
| Plates | EXR or ProRes (per spec) | The actual camera footage |
| Reference Cut | H.264 / DNxHD with TC burn-in | Context for each shot |
| Turnover EDL / AAF | EDL or AAF | Conform reference |
| VFX Notes | PDF or spreadsheet | Shot-specific creative direction |
| Camera Data | ALE / CSV | Lens, camera, ISO, WB per shot |
| LUT / OCIO Config | .cube / .ocio | Color reference for viewing plates |
| HDRI | EXR | Lighting reference (per scene if available) |
| Survey Data | Spreadsheet / diagrams | Scale and position reference |
| Clean Plates | EXR or same format as plates | Background coverage |

### VFX Pull List — Required Fields
At minimum, every shot entry needs:
- **VFX Shot #** (e.g., `SC015_010_VFX`)
- **Scene & Shot** (from script / editorial)
- **Reel/Camera** (source camera)
- **Cut In** (timecode or frame # in source clip)
- **Cut Out** (timecode or frame # in source clip)
- **Handle In** (cut in − N handles)
- **Handle Out** (cut out + N handles)
- **Duration** (with handles)
- **Tier** (T1–T4)
- **VFX Description** (brief)
- **Assigned Vendor**
- **Delivery Date**

## Handles

**Handles** are extra frames included before and after the editorial cut point, giving
compositors room to work without hard transitions at the frame boundary.

### Standard Handle Lengths
| Context | Handle Length |
|---------|--------------|
| Standard VFX shots | 8–16 frames head and tail |
| Complex/long shots | 16–24 frames |
| MoCap / performance-matched shots | 24+ frames |
| Vendor-specific requirement | Always confirm — some facilities specify |

### Why Handles Matter
- Compositors need frames to stabilize motion, build smooth transitions into/out of work
- If handles are too short, any editorial recut that extends the shot requires a new plate pull
- **Always err on the side of longer handles** — storage is cheaper than re-pulling plates

### Handles and Frame Count Reporting
On the pull list, always report:
- **Cut duration** (handles excluded) — what editorial is currently using
- **Plate duration** (handles included) — what you're delivering to vendor
- Frame rates must match — if editorial is at 23.976, plates must be at 23.976

## Offline vs Online Editorial

### Offline Editorial
- Low-resolution proxies (ProRes LT, DNxHD 115, H.264)
- Where creative editorial decisions are made — cut, pacing, structure
- No VFX finals here — postvis / pre-comp used as placeholders
- **Output:** EDL/AAF for online conform

### Online Editorial
- Full-resolution, full-quality files
- VFX finals replace postvis/pre-comp
- Color graded in DaVinci Resolve, Baselight, or similar
- Where the final master is created

### Picture Lock
- The moment editorial declares the cut is final
- VFX shots added after picture lock = change orders (additional cost, schedule impact)
- VFX Supervisor should track the "near-lock" shot list to prepare for final deliveries
- Establish a **soft lock** (all sequences locked except final act) as an intermediate milestone

## EDL, AAF & XML Conform

### EDL (Edit Decision List)
- Simple text-based list of edit decisions
- Contains: event #, reel/tape name, edit type, timecodes (record in/out, source in/out)
- CMX3600 format is the industry standard
- **Limitation:** EDL only handles one video track; no nested sequences; no effects metadata

### AAF (Advanced Authoring Format)
- Richer than EDL — contains multiple tracks, metadata, audio, effects
- Generated by Avid Media Composer; readable by most NLEs and conform systems
- **Best for:** Complex multi-track conforms; retaining audio and metadata
- **VFX use:** Pull lists can be generated from AAF more reliably than EDL

### XML (FCP / Resolve XML)
- Final Cut Pro X / Premiere Pro native format
- Preserves project structure, effects, nested sequences
- DaVinci Resolve can import FCP XML for conform
- **VFX use:** Increasingly common for Premiere-based episodic pipelines

### Conform Workflow
1. Editorial exports **AAF or XML** at picture lock
2. Online/conform system reconforms to **full-res media**
3. VFX finals are **replaced** into the timeline (by shot name/timecode match)
4. Colorist grades the conformed timeline
5. Output: final master

## VFX Reference Cuts

### Purpose
A reference cut is a low-res H.264 or DNxHD export of the current editorial cut,
burned in with timecode, scene/shot/take, and frame numbers.

### What to Include in Reference Cut
- **Timecode burn-in** (record TC and source TC)
- **Scene/shot/take** from camera report
- **VFX shot numbers** where known
- Frame count from head of shot
- Show name, reel number, date generated

### Reference Cut Delivery to Vendors
- Deliver at every turnover — even small updates
- Include as an **H.264 for distribution** + DNxHD for archival reference
- Label clearly with version: `SHOW_VFXRef_Reel1_v003_20241015.mov`
- Vendors use the reference cut to understand cut context, timing, adjacent shots

## Plate Delivery to Vendors

### Delivery Specs — Standard VFX Plates
| Parameter | Specification |
|-----------|--------------|
| Format | EXR sequence (preferred) or ProRes 4444 |
| Bit depth | 16-bit half-float EXR |
| Compression | ZIP or ZIPS (lossless) |
| Colorspace | Camera native log OR linear/ACEScg — confirm with vendor |
| Resolution | Full camera native (no downscale) |
| Handles | As per pull list (8–16 frames min) |
| Frame naming | `SC015_010_Cam_A_RawLogC4_####.exr` (see naming section) |
| Delivery method | Aspera, Signiant, or facility FTP — never consumer cloud |

### What NOT to Deliver
- **Never deliver with LUT baked in** — vendor must work with clean log or linear
- **Never deliver proxy resolution** as hero plates
- **Never transcode RAW to ProRes then deliver** as RAW — clarify if RAW is needed

### Delivery Verification
Before sending:
- [ ] Correct frame count (handle in to handle out)
- [ ] Correct colorspace (confirmed with vendor)
- [ ] No missing frames in sequence
- [ ] MD5 checksums generated
- [ ] Manifest / shotlist included

## VFX Finals Delivery

### Finals Delivery Specs from Vendors
Typically two deliveries per shot:
1. **EXR (multi-layer)** — scene-linear, all AOVs, for DI conform
2. **Proxy (ProRes or H.264)** — for review and editorial reference

### EXR Finals Requirements
| Parameter | Specification |
|-----------|--------------|
| Format | Multi-layer EXR |
| Bit depth | 16-bit half-float (32-bit float for VFX shots with > 1.0 linear values) |
| Colorspace | ACEScg scene-linear (or project-specified) |
| Layers | Beauty + any split elements (see below) |
| Compression | ZIP or ZIPS |
| Alpha | Premultiplied |

### EXR Layers — Standard Delivery
| Layer | Description |
|-------|-------------|
| `beauty` (RGB) | Final composite, no background |
| `diffuse` | CG diffuse pass |
| `specular` | CG specular pass |
| `reflection` | CG reflection pass |
| `emission` | CG self-illumination |
| `shadow` | Shadow catcher pass |
| `alpha` (A) | Matte for the VFX element |
| `holdout` | Holdout matte for integration |

Confirm exact pass requirements with DI/finishing at project start.

## File Naming Conventions

### Industry-Standard VFX Naming

**Shot identifier format:**
`[SHOW]_[SCENE]_[SHOT]_[VERSION]_[FRAME].ext`

Example: `PROJ_SC015_010_v025_0001.exr`

**Expanded format for plates:**
`[SHOW]_[SCENE]_[SHOT]_[CAMERA]_[COLORSPACE]_[FRAME].ext`

Example: `PROJ_SC015_010_A_LogC4_0001.exr`

### Key Conventions
- Scenes: zero-padded to 3 digits (`SC015`)
- Shots: zero-padded to 3 digits (`010` = shot 10 within scene)
- Versions: `v001`, `v002` — never `v1` or `V001`
- Frames: **4-digit zero-padded** (`0001`) minimum; use `%04d` in Nuke/shell
- **No spaces** in filenames — use underscores
- **No special characters**: no `!@#$%^&*()`
- **Case consistent** — establish whether project is all-lowercase or CamelCase and enforce

### Version Bumping Rules
- Increment version on **every delivery to DI/editorial**, even minor changes
- Never overwrite a previous version — all versions must be preserved
- Keep a **version notes log** per shot: what changed in each version

## Finishing & Online Conform Integration

### VFX Supervisor Role in Finishing
- **Review VFX finals on a calibrated display** before delivery to DI — not just on a laptop
- Confirm colorspace of finals matches DI's expectation
- Attend **VFX pass in DI** — review each shot on the grading monitor, confirm integration
- Flag any shots needing **DI adjustment** (comp that doesn't match grade, clean-up needed)

### Common Finishing Issues from VFX
| Issue | Cause | Resolution |
|-------|-------|-----------|
| VFX shot looks different from surrounding cuts | Different colorspace or ODT applied in comp | Confirm scene-linear delivery; let colorist handle ODT |
| Grain/noise mismatch between VFX and plate | VFX shot is too clean | Add grain in comp to match camera noise profile |
| Edge artifacts visible at 4K | Comp was checked at 2K only | Always QC at deliverable resolution |
| VFX elements too bright/dark vs grade | Colorist graded around old postvis; finals different | Attend grade session for VFX shots |
| Missing alpha/incorrect premultiplication | Wrong comp output settings | Confirm premult state with DI before delivery |
